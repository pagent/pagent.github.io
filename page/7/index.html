rekonqのデフォルト検索エンジンを設定出来ない

2015年8月21日

Arch LinuxでサブのブラウザとしてMidoriを使っているが、お気に入りが消えてしまったりと調子が悪かったので、rekonqを入れてみた。

サイドバーは有る方が好みなので、レンチアイコンの設定からPanels→Bookmarks Panelをチェック。
広告除去は、Tools→Ad block。
EasyListだけでは依然、広告が邪魔な状態だったので、EasyList without element hidingも有効にした。

快適に使えるようになったかと思ったが、起動の度に、デフォルトの検索エンジンを設定するよう、上部に注意表示される。

you do not have a default search engine set. without it …

確かにアドレスバーに語句を入力しても検索出来ない。
注意表示の「Set it」をクリックしてもなにも起こらないし、設定のSearch Enginesの箇所は、モジュールが無いようで灰色、設定出来ない。

Konquerorをインストールしたら良いという情報があったので、インストールしてみた。
cannot set the default search engine in rekonq 1.0 / Applications & Desktop Environments / Arch Linux Forums

検索エンジンを設定出来るようになった。
Konquerorは使わないようだったら削除してしまっても良いようだ。

Konquerorをアンインストールすると、検索エンジンのモジュールも削除されてしまうようだ。

右クリックからの検索を拡張するFirefoxアドオン「Context Search」が便利

2015年8月15日

Firefoxで右クリックからの検索は便利だが、検索エンジンがデフォルトのものに固定されてしまっている。
普段はGoogleを使っているが、安全のためStartPageやDuckDuckGoを使いたい時、またツイッター検索をしたい時などがある。そんな時、キーワードをいちいち検索欄にコピペするのはちょっと面倒。

そんな人にお勧めしたいアドオンがContext Search

インストールすると、右クリックの「”〜” をGoogleで検索(s)」などが「”〜” を検索(s)」になり、種々の検索エンジンがリスト表示される。
特に設定の必要も無し。

ちょっとしたことだが、とても便利。

架空のメールアドレスを使う

2015年7月9日

ネットを利用していて、例えばメール欄が必須になっているブログにコメントしたい時など、これまでは正直に実際のメールアドレスを記載していた。（メインのではなく、サブのアドレスを使うことが多かったが）
だが、よく考えてみると、返信が必要のない場合は実際のものではなく、架空のメールアドレスを使っても問題なさそうだ。

存在するかもしれないメールアドレスを適当に使ってしまうと迷惑を掛けてしまうかもしれないが、３つのサンプル用ドメイン（example.com、example.org、 example.net）なら、大丈夫なはず。

そう考えて、適当な架空のメールアドレスとしてmail@example.netを使うようにした。

ツイッター検索でURL付きのツイートが表示されない

2015年6月30日

自分のツイートだけが表示されない

ニュース記事やブログなどに設置してあるツイート数表示をクリックした時に表示されるページ（URLをツイッター検索した結果のページ）に自分のツイートだけが表示されなくなっていた。「トップ」から「ライブ」のタブに切り替えても表示されない。

これではツイートを見てもらう機会が減ってしまい困るので、下記ページから問題の報告を行った。
Twitterヘルプセンター

「問題の詳細」の箇所は英文で書いた方が良いらしいので、見様見真似で下記のようにした。

Regarding: Search/Trends
Subject: I’m not a spam!!
Description of problem:
My tweets that contain URLs are not displayed in search results. Can you permit me to display my tweets in the search? I use Twitter generally. I’m not spam. Please clear your information about spam. Kind regards.

「返信できませんが、いただいた情報はTwitterの改善に役立たせていただきます。」とのことだったが、１日で改善された。

自分のブログ記事のツイッター検索の結果が１件しか表示されない

その後、別で書いている自分のブログでのことだが、上記同様ツイート数表示をクリックした時に表示されるページ（URLをツイッター検索した結果のページ）で、何人かツイートしてくれている人がいるのに、ツイートが１件しか表示されない状態になっていた。「トップ」でも「ライブ」でも１件のみ。誰のツイートが表示されるかは、リツイート数の多いツイートだったり、最新のツイートだったり。
これではツイートしてくれる人も意欲を削がれるに違いない。

上記の「Twitterヘルプセンター」から日本語で報告してみたが、しばらく待っても変化が無い。
再度下手くそな英語で報告してみた。

Regarding: Search/Trends
Subject: About search result
Description of problem:
Reasntly, several tweets exist that contain the URL(http://pagent.seesaa.net/article/421338174.html). But only one tweet is displayed in search result, even in “Live” tab.
Can you adjust this probrem?

１〜２時間ほどすると、ほぼすべてのツイートが表示される状態に改善した。

英語が苦手でも、やってみるものだと実感した。

GitHubが「ギットハブ・ジャパン」を設立したそうだ

2015年6月4日

ITmediaのニュースによると、Gitによるバージョン管理サービスの老舗「GitHub」が日本法人「ギットハブ・ジャパン」を設立したそうだ。
GitHubが日本法人「ギットハブ・ジャパン」設立　日本語テクニカルサポートで法人向け導入を促進 (1/2) - ITmedia ニュース

ギットハブ・ジャパンは企業向けのツール「GitHub Enterprise」の販売とサポート提供が主な目的のよう。
“github”を検索してみたら、すぐにGitHub Japanのサイトが見つかった。
http://github.co.jp/

どうやらすぐにgithub.comに日本語ページが用意されるということではないようだが、英語サイトの「GitHub Enterprise」は”github.com”のサブドメイン”enterprise.github.com”で提供されているようだし、もしかするとそのうちgithub.comを日本語で利用出来るようになるのだろうか？
（類似のサービスを提供している「Bitbucket」はすでに英語だけでなく、日本語、ドイツ語、フランス語、ロシア語、中国語、ヒンズー語など、計９つの言語で利用可能になっている。）

自分はIT関係には疎くて、GitHub Pagesをブログ代わりに利用させてもらっているだけだが、もしgithub.comを日本語で利用出来るようになれば、今後個人の日本人利用者もさらに増えるのかもしれない。

少しでも興味のある人は早めに登録しておいた方が、気に入ったアカウントを取得しやすいかもしれませんね。
